# Техническое задание: LEDS — система автоматического проектирования светодиодной засветки

**Версия:** 1.0  
**Статус:** Проектирование  
**Целевая аудитория:** технологи, менеджеры, конструкторы рекламно-производственных компаний  

---

## 1. Назначение и область применения

### 1.1 Назначение

Коммерческое настольное приложение **LEDS** (LED Engineering & Design System) для автоматического и полуавтоматического проектирования внутренней засветки:

- плоских и объёмных вывесок;
- объёмных букв (channel letters);
- световых коробов;
- фигурных элементов сложной геометрии.

Система принимает векторные макеты из CorelDRAW (SVG, DXF), строит геометрическую модель изделия, размещает светодиодные модули по интеллектуальному алгоритму, моделирует равномерность засветки и формирует комплект производственной и коммерческой документации.

### 1.2 Целевые пользователи

| Роль | Задачи |
|------|--------|
| Менеджер | Импорт макета, выбор модуля, КП, согласование с клиентом |
| Технолог | Тонкая настройка размещения, проверка рисков, ТЗ |
| Конструктор | Глубина, борт, крепления, схема подключения |
| Администратор | Справочники модулей, цены, шаблоны документов |

### 1.3 Ограничения и допущения

- Вход: **SVG 1.1** (приоритет), **DXF R12–R2018** (полилинии, дуги, сплайны — поэтапно).
- Единицы: мм (с возможностью пересчёта из дюймов/px при импорте).
- Плоская проекция: размещение в плоскости «лицевой» грани; глубина — параметр расчёта, не 3D-CAD.
- ОС: Windows 10/11 (приоритет), далее macOS.
- Офлайн-first: расчёты локально; облако — опционально (лицензии, обновления справочников).

### 1.4 Критерии приёмки продукта (уровень «реальное производство»)

1. Корректная обработка **внешних контуров и отверстий** (буквы O, R, B и т.д.).
2. Учёт **ширины борта** и **глубины** канала/короба при допустимости размещения.
3. Размещение без примитивной равномерной сетки; адаптация к **тонким штрихах** и **острым углах**.
4. Тепловая карта с выделением зон **недосвета / пересвета / пятен**.
5. Расчёт: количество модулей, поток, мощность, БП, ориентировочная себестоимость.
6. Экспорт PDF: ТЗ, спецификация, схема подключения, КП.
7. Три режима: автоматический, полуавтоматический, экспертный.
8. Время расчёта типовой буквы (до 500 мм): **< 3 с** на ПК среднего класса (целевое).

## 3. Нефункциональные требования

### 3.1 Производительность

- Импорт SVG до 10 МБ: < 2 с.
- Построение medial axis / distance field для контура до 10⁴ сегментов: < 1 с.
- Рендер heatmap 2048²: < 500 мс (GPU при наличии).
- Автосохранение проекта каждые 60 с.

### 3.2 Надёжность

- Атомарное сохранение проекта (.leds — ZIP + JSON + бинарные слои).
- История undo/redo (минимум 50 шагов для размещения).
- Валидация входных файлов с понятными сообщениями на русском.

### 3.3 Безопасность

- Подписанные пакеты обновления справочников.
- Локальное хранение; передача в облако только по явному согласию.

### 3.4 Локализация

- UI: русский (по умолчанию), английский (фаза 2).
- Документы: шаблоны RU.

## 4. Модель данных проекта (логическая)

```
Project
├── metadata (название, клиент, автор, дата)
├── importSources[] (svg/dxf paths, hash)
├── geometry
│   ├── contours[] (rings: outer + holes)
│   └── layers[]
├── productParams (depth, rimWidth, diffuser, material)
├── moduleCatalogRef (version)
├── placements[] (moduleId, x, y, angle, fixed, chainId)
├── powerPlan (groups, PSU assignments)
├── simulationSettings (gridRes, thresholds)
└── documents[] (generated PDF refs)
```

## 5. Сценарии использования (кратко)

### UC-01 Автоматический расчёт вывески

1. Менеджер импортирует SVG.
2. Система строит контуры и предлагает параметры (глубина, борт).
3. Пользователь выбирает модуль из библиотеки.
4. Запуск «Авторазмещение» → карта засветки.
5. При предупреждениях — переход в полуавтомат или смена модуля.
6. Экспорт ТЗ + КП.

### UC-02 Тонкая доработка технологом

1. Открытие проекта в режиме 2.
2. Фиксация критичных модулей у острых углов.
3. Добавление модуля в зону недосвета.
4. Пересчёт карты и мощности.
5. Экспорт схемы монтажа.

### UC-03 Экспертная ручная укладка

1. Режим 3: пустое размещение или очистка авто.
2. Ручная укладка с привязками.
3. Проверка только светотехники и БП.

## 6. Глоссарий

| Термин | Определение |
|--------|-------------|
| Контур (ring) | Замкнутый полигон/кривая; outer — внешний, hole — отверстие |
| Допустимая зона | Область внутри контура минус offset(борт + половина габарита модуля) |
| Medial axis | Осевой скелет формы — линия равноудалённая от границ |
| Пятно модуля | 2D-распределение относительной освещённости от одного LED |
| Переуплотнение | Локальный избыток вклада соседних пятен выше порога |

## 7. Вне scope v1.0
